Privacy Policy for ArchCraft GPT

Last updated: 2025-09-20

Overview

ArchCraft GPT helps you look up Arch Linux packages, AUR packages, and Linux kernel releases. When you ask a question, the GPT may call public APIs listed below to fetch data needed to answer. No personal data is collected, stored, or sold by the creator of this GPT.

Public APIs Used

  • archlinux.org (official repositories)
  • aur.archlinux.org (Arch User Repository)
  • kernel.org (Linux kernel releases)
  • geo.mirror.pkgbuild.com (Arch mirrors / pkgbuild content)

APIs & Endpoints Implemented

archlinux.org

  • GET https://archlinux.org/packages/search/json/?name=<term>&repo=<repo?>&arch=<arch?>
  • GET https://archlinux.org/packages/{repo}/{arch}/{pkgname}/json/

aur.archlinux.org (AUR RPC v5)

  • GET https://aur.archlinux.org/rpc/?v=5&type=search&arg=<term>
  • GET https://aur.archlinux.org/rpc/?v=5&type=info&arg[]=<pkgname>
  • GET https://aur.archlinux.org/rpc/?v=5&type=multiinfo&arg[]=<pkg1>&arg[]=<pkg2>â€¦

www.kernel.org

  • GET https://www.kernel.org/releases.json

geo.mirror.pkgbuild.com

  • Static content / file listings as needed for package metadata references.

What Data Is Sent

The GPT only sends the minimum query parameters required to perform your request, for example:

  • Package names or partial names (e.g., hyprland, proton-ge-custom)
  • Repository / architecture when relevant (e.g., extra/x86_64)

No personally identifiable information (PII) is sent by this GPT. No API keys or authentication are used.

Data Storage & Retention

  • The GPT does not store any personal data.
  • Requests to public APIs are transient and used only to fulfill your query.
  • No server-side databases are maintained by the GPT creator for this project.
